Build an event-triggered onboarding email sequence for both publishers and authors using Resend. All emails must match the Inkstain brand — parchment #f5f2eb background, ink #1B2A3B, amber #c8956b, Playfair Display headings, EB Garamond body, clean and minimal.

Infrastructure first
Add a scheduled_emails table to track what has and hasn't been sent:
sqlCREATE TABLE IF NOT EXISTS scheduled_emails (
  id SERIAL PRIMARY KEY,
  recipient_email TEXT NOT NULL,
  recipient_type TEXT NOT NULL, -- 'publisher' or 'author'
  recipient_id TEXT NOT NULL,
  email_key TEXT NOT NULL, -- unique identifier for each email in sequence
  triggered_by TEXT NOT NULL, -- 'account_created', 'first_contributor', 'first_certificate'
  sent_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE UNIQUE INDEX IF NOT EXISTS idx_scheduled_emails_dedup 
  ON scheduled_emails (recipient_id, email_key);
The unique index prevents any email from being sent twice to the same recipient.

Sending logic
Create a helper function sendOnboardingEmail(recipientId, recipientType, emailKey, triggeredBy, templateData) that:

Checks scheduled_emails for existing record with same recipient_id + email_key
If already sent — skip silently
If not sent — render the email template, send via Resend, insert record into scheduled_emails with sent_at = NOW()

Call this function from the appropriate places:

Publisher signup handler → publisher_welcome
Author signup handler → author_welcome
Author association with publisher (invite link signup) → check contributor count, fire publisher_first_contributor to the publisher if this is their first
Certificate generation handler → author_first_certificate
Certificate verify handler → publisher_first_verify to publisher + author_certificate_verified to author (if publisher_id known)

All email sending is fire-and-forget — wrap in try/catch, never block the main request.
